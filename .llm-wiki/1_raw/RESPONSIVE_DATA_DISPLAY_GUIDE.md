# Responsive Data Display & Layout Stability Guide

## ปัญหาที่พบเจอ (The Problem)
เมื่อหน้า Dashboard หรือหน้ารายการเคสต้องแสดงข้อมูลที่มีความยาวไม่แน่นอน (เช่น ชื่อประเภทงาน, ตัวเลขสรุปยอด) การใช้ CSS Flexbox หรือ Grid แบบปกติมักจะเจอปัญหา:
1. **Grid Stretch / Flex Blowout**: คอลัมน์กว้างไม่เท่ากัน หรือโดนเนื้อหาดันจนทะลุหน้าจอ (โดยเฉพาะบนจอมือถือ)
2. **Missing Scrollbars**: การซ่อน Scrollbar ทำให้ผู้ใช้ไม่รู้ว่าสามารถเลื่อนดูข้อมูลได้ 
3. **Card Consistency**: ขนาดของการ์ดสรุปข้อมูล (Metric Cards) ดูไม่สม่ำเสมอเมื่อมีข้อความจำนวนมาก

## แนวทางแก้ไข (Best Practices)

### 1. การจัดการ Grid และ Text Truncation
การใช้ Grid ใน Tailwind ต้องระวังเรื่อง `minmax` ของเนื้อหา ถ้าไม่ใส่ข้อจำกัด, ข้อความยาวจะดัน Grid Column ให้ขยายจนทำให้ Grid พัง
- **วิธีแก้**: ใส่ `min-w-0` ให้กับ Parent Container ที่อยู่ใน Grid หรือ Flex และใส่ `truncate` (ซึ่งก็คือ `overflow: hidden; text-overflow: ellipsis; white-space: nowrap;`) ให้กับ Text Element 
- **ตัวอย่าง**:
  ```tsx
  <div className="grid grid-cols-2 sm:grid-cols-3 xl:grid-cols-5 gap-4">
    <div className="glass-card min-w-0">
      <p className="truncate min-w-0">{longText}</p>
    </div>
  </div>
  ```

### 2. การแยกแยะความหมายของคำว่า "เคส (Cases)" และ "ชิ้นงาน (Units)"
บ่อยครั้งที่ผู้ใช้งานสับสนระหว่างจำนวนตั๋วงานและจำนวนสินค้าในตั๋ว
- **Cases (เคส)**: คือ 1 ตั๋วงาน (เช่น มี 1 การแจ้งเคลม)
- **Units (ชิ้นงาน/ปริมาณ)**: คือปริมาณย่อยที่อยู่ในเคส (เช่น 1 เคส อาจจะเคลมสินค้า 100 ชิ้น)
- **การสื่อสารใน UI**: ควรมีคำอธิบาย (Tooltip หรือ Helper Text) ในส่วนของกราฟให้ชัดเจนว่า "นับตามจำนวนเคส" หรือ "นับตามปริมาณชิ้นงาน"

### 3. การยุบรายละเอียดในตาราง (Data Collapsing)
หากเคสหนึ่งๆ มีรายการ (Items) หลายรายการ:
- ไม่ควรนำทุกรายการมาแสดงทั้งหมดในตารางหลักทันที เพราะจะกินพื้นที่แนวตั้งมหาศาล
- ควรแสดงแค่ "รายการแรก" พร้อมปุ่ม `+x รายการ` (เช่น `+2 items`)
- หรือใช้เทคนิค **Accordion/Expandable Row** เพื่อให้ผู้ใช้งานกดขยายดูเมื่อสนใจ
- หรือใช้ **Modal** สำหรับแสดงรายละเอียดฉบับเต็มเมื่อกดปุ่ม "ดูรายละเอียด"

### 4. การจัดการ Scrollbar สำหรับข้อมูลแนวนอน
บนหน้าจอมือถือ หากข้อมูลตารางหรือไอเทมเป็นแนวนอน การปล่อยให้มี Scrollbar จะดีกว่าการบีบ (Squeeze) ทุกคอลัมน์ให้เล็กจนอ่านไม่ออก
- ใช้ `overflow-x-auto` และตั้งค่าการแสดงผล Scrollbar ให้สวยงามและมองเห็นได้ 
- สำหรับ Tailwind สามารถเพิ่มคลาสปรับแต่งแต่ง Scrollbar หรือเว้นขอบ (Padding) ด้านขวาให้มีพื้นที่สำหรับการเลื่อน

## การนำไปใช้ในโปรเจกต์ QSMS
- เราได้ปรับใช้คลาส `lg:grid-cols-5` หรือ `xl:grid-cols-5` สำหรับ MetricCards เพื่อให้รองรับขนาดหน้าจอต่างๆ โดยไม่บีบแคบจนเกินไปในหน้าจอ Tablet
- ใช้ `min-w-0` และ `truncate` ภายในคอมโพเนนต์ MetricCard
- ปรับสไตล์ของ Panel กราฟต่างๆ ให้เป็น `glass-panel` อย่างสม่ำเสมอ ลดการใช้ `bg-white` ที่อาจทำให้ดูหลุดโทน (Theme Inconsistency)
