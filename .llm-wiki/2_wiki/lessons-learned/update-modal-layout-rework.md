## 1. ปัญหาการเลื่อนของหน้าจอ (Viewport Jumping) & React Portals
*   **อาการ**: เมื่อผู้ใช้คลิกดูพรีวิวรูปภาพขนาดใหญ่ (Lightbox) หรือเปิดโปรแกรมแก้ไขภาพ (Image Editor) หน้าจอเบราว์เซอร์จะดีด/เลื่อนกลับไปด้านบนสุด (Scroll to top) ทำให้เสียตำแหน่งการทำงานเดิม
*   **สาเหตุ**: กล่อง Modal พรีวิวถูกเรนเดอร์อยู่ภายใต้คอมโพเนนต์แม่ที่มีการอนิเมชันหรือใช้ CSS Transforms (เช่น คลาสของ `framer-motion` หรือ Next.js Tabs) ส่งผลให้เบราว์เซอร์ตีความ `position: fixed` ของ Lightbox เสมือนเป็น `position: absolute` อิงตามคอมโพเนนต์แม่ที่ครอบอยู่
*   **แนวทางการแก้ไข**: 
    *   นำไลบรารี `createPortal` จาก `react-dom` มาใช้เรนเดอร์ Lightbox และ ImageEditor ออกไปนอก Stacking Context ของคอมโพเนนต์แม่ โดยนำไปต่อที่ `document.body` โดยตรง
    *   ใช้สถานะ `mounted` (ผ่าน `useEffect`) เพื่อป้องกันความขัดแย้งของ Server-Side Rendering (SSR Hydration Mismatch) ใน Next.js (เนื่องจากเซิร์ฟเวอร์ไม่มี `document`)

## 2. ปัญหาสามประสานของหน้าจอขนาดเล็ก (Responsive & Double Scrollbars)
*   **จุดบกพร่อง**:
    1.  หน้าจอโหมดดูตัวอย่าง (View Mode) ตัดแยกคอลัมน์ซ้าย-ขวาที่ขนาด `md` (768px) แต่โหมดแก้ไข (Edit Mode) ตัดที่ `lg` (1024px) ทำให้ช่วงหน้าจอขนาด 768px ถึง 1024px ฝั่งขวาแคบมากจนช่องกรอกข้อมูลเบียดเสียด
    2.  หน้าต่างฝั่งซ้ายและขวาต่างมี `overflow-y-auto` ของตัวเอง ทำให้เกิดแถบเลื่อนแนวตั้ง (Scrollbar) คู่กัน 2 เส้น ดูอึดอัดและไม่สวยงาม
    3.  ตารางรายการวัสดุ (Materials Table) ในโหมด View ไม่มีคลาสคุมความกว้างขั้นต่ำและกรอบเลื่อนแนวนอน ส่งผลให้ปุ่มเลือกและกล่องจำนวนถูกบีบแบนจนไม่สามารถพิมพ์ได้
*   **แนวทางการแก้ไข**:
    *   **ปรับจุดสลับแถวเป็น `lg` (1024px)**: ทำให้คอลัมน์ซ้อนกันแนวดิ่งบนแท็บเล็ตและอุปกรณ์ขนาดเล็ก มีพื้นที่แสดงผลกว้างขึ้น
    *   **ตรึงคอลัมน์ฝั่งขวา (Fixed Right Column)**: ปรับคอลัมน์ฝั่งขวาเป็น `overflow-hidden` ร่วมกับ Flex Layout เพื่อปิดแถบเลื่อนแนวตั้ง ทำให้ข้อมูลสถานะ การเงิน และเวลาทำงานนิ่งอยู่กับที่ มองเห็นได้ครบถ้วนโดยไม่ต้องเลื่อนหน้าจอ
    *   **จำกัดความสูงตารางวัสดุ (Capped Height & Scroll)**: ครอบตารางวัสดุทั้งสองโหมดด้วยความสูงสูงสุด `max-h-[220px]` และมีคุณสมบัติ `overflow-auto custom-scrollbar min-w-[500px]` หากรายการวัสดุเพิ่มขึ้น จะสามารถเลื่อนในกล่องตารางได้ด้วยตัวเอง โดยไม่ล้นกรอบหน้าจอหลักและไม่บีบช่องกรอกข้อมูล

## 3. กับดักคลาส `uppercase` ของ Tailwind
*   **อาการ**: เมื่อปรับแก้ข้อความป้ายกำกับภาษาอังกฤษให้เป็น Title Case เช่น `แม่พิมพ์ (Mold)` หรือ `สายการผลิต (Line)` แล้ว แต่หน้าเว็บจริงยังคงแสดงผลเป็นตัวพิมพ์ใหญ่ทั้งหมด (`MOLD`, `LINE`) เสมอ
*   **สาเหตุ**: มีคลาส `uppercase` ของ Tailwind CSS ถูกฝังอยู่ใน CSS class list ของแถบหัวเรื่อง (`className="... uppercase ..."` ) ซึ่งคลาสนี้จะเปลี่ยนข้อความทุกตัวอักษรให้เป็นตัวใหญ่โดยอัตโนมัติที่ระดับของเบราว์เซอร์
*   **แนวทางการแก้ไข**: ถอนคลาส `uppercase` ออกจากป้ายกำกับหัวเรื่องในซอร์สโค้ด เพื่อให้ฟอนต์เคารพรูปแบบการเขียนอักษรภาษาอังกฤษผสมปกติ

## 4. การจัดการความคลีนและการตัดฟิลด์ (UX Clean-up)
*   **การเอา "วิธีแก้ไขปัญหา (Resolution)" ออก**:
    *   เนื่องจากกระบวนการ rework ยุคใหม่เปลี่ยนเน้นบันทึกสถานะและเบิกวัสดุเป็นหลัก จึงทำการถอดถอนช่องกรอกและแสดงผลของวิธีแก้ไขปัญหาออกทั้งหมด เพื่อเคลียร์พื้นที่ฝั่งขวาให้กระชับ
*   **การย้ายแถบส่งออกเอกสาร**:
    *   ย้ายปุ่มดาวน์โหลดรายงาน (**PNG**, **Excel**, **PDF**) จากเดิมที่อยู่ด้านล่างสุดของหน้าจอ (Footer) ขึ้นไปอยู่มุมขวาบนในส่วนของ Header ของ Modal ในรูปแบบไอคอนปุ่มทรงกลม Apple style ช่วยให้เรียกใช้งานได้รวดเร็วและเป็นสัดส่วน
