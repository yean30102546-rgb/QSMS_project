## 1. Summary
สรุปแนวทางการจัดการพฤติกรรมพังของ AI Agent (เดา, โกหก, ทำเกิน, ลืม) ผ่านการติดตั้งกฎและเฟรมเวิร์กควบคุม 4 ชั้น (Guardrails, Memory, Spec-driven, Personalization) ลงในไฟล์การกำหนดค่าของเอเจนต์ (เช่น `CLAUDE.md` หรือ `AGENTS.md`) เพื่อให้เอเจนต์ทำงานอย่างมีประสิทธิภาพและไม่หลุดขอบเขตงาน

---

## 2. Key Details — 4 ชั้นของการควบคุมเอเจนต์

### 🛡️ ชั้น 1: Guardrails (กฎกันพฤติกรรมพัง)
- **NO MAGIC**: หากไม่รู้ข้อมูลจริง ห้ามเดาเด็ดขาด รวมถึงห้ามเดาเส้นทางไฟล์ (Paths) หากหาไม่พบให้ถามมนุษย์ทันที
- **VERIFY BEFORE DONE (สำคัญที่สุด)**: ห้ามรายงานว่า "เสร็จแล้ว" (Done) หากยังไม่ได้ทดสอบรันจริงและไม่มีหลักฐานยืนยันความถูกต้อง
- **DISSENT**: ก่อนลงมือปรับปรุงระบบขนาดใหญ่ เอเจนต์ต้องทักท้วงและอธิบายความเสี่ยง ผลกระทบ และแผนการยกเลิก/ย้อนกลับ (Rollback) ให้มนุษย์พิจารณาก่อน
- **SCOPE DRIFT**: ห้ามแอบ Refactor โค้ดทั้งโมดูลในขณะที่ได้รับคำสั่งให้แก้ไขบั๊กเฉพาะจุด
- **R0 / R1 / R2 (การประเมินสิทธิ์ตัดสินใจตามความเสี่ยง)**:
  - **R0 (ความเสี่ยงสูง/ถอยกลับไม่ได้)**: หยุดและถามมนุษย์ก่อนเสมอ
  - **R1 (ความเสี่ยงปานกลาง/ถอยกลับยาก)**: ดำเนินการแล้วรายงานผลทันที
  - **R2 (ความเสี่ยงต่ำ/ถอยกลับง่าย)**: ดำเนินการได้ทันทีโดยไม่ต้องรออนุมัติ

### 🧠 ชั้น 2: Memory (ระบบป้องกันการลืมข้าม Session)
- จัดทำไฟล์ `MEMORY.md` ที่ root ของโปรเจกต์
- กำหนดให้ AI เป็นผู้เขียนอัปเดตบทเรียนความผิดพลาดลงไฟล์นี้ด้วยตัวเองหลังจบงานที่มีข้อผิดพลาด
- โครงสร้างของแต่ละรายการในบันทึก (Entry) ประกอบด้วย 3 ช่อง:
  1. **เกิดอะไรขึ้น** (Symptom/Issue)
  2. **ทำไมถึงเกิด** (Root Cause)
  3. **ครั้งหน้าต้องทำอย่างไร** (Correct Behavior/Actionable Instruction)

### 💾 ชั้น 3: Spec-driven (การทำ Savepoint ป้องกันลืมเป้าหมายระหว่างแชท)
- สร้างไฟล์สเปกระหว่างทาง เช่น `spec.md` หรือ `task.md` เพื่อให้เอเจนต์บันทึกสถานะปัจจุบัน (Current State), การตัดสินใจสำคัญ และขั้นตอนถัดไปที่จะทำ
- กำหนดกฎให้เอเจนต์อ่านเอกสารนี้ทุกครั้งก่อนเริ่ม Session ใหม่ และอัปเดตข้อมูลทุกครั้งหลังเสร็จสิ้นแต่ละภารกิจย่อย เพื่อให้สามารถกดล้างแชท (`/clear`) ได้โดยไม่เสียบริบท

### 👤 ชั้น 4: Personalization (การดึงวิธีคิดสไตล์คุณ)
- คัดลอก Work Pattern, โทนเสียงการตอบกลับ (Voice), และ Trigger คำสั่งลัดจากสไตล์ส่วนตัวของมนุษย์ที่คุยด้วยบ่อยๆ มาใส่ไว้ใน `AGENTS.md` เพื่อสร้างความคุ้นเคยและเพิ่มประสิทธิผลการทำงานร่วมกัน
