# คู่มือการใช้งานระบบ QSMS Rework Management System (ฉบับละเอียด)

ระบบนี้ถูกออกแบบมาเพื่อจัดการและติดตามกระบวนการทำงาน Rework สินค้า โดยเชื่อมต่อข้อมูลระหว่างหน้าเว็บ (Next.js/React) และฐานข้อมูล Supabase Database ควบคู่กับ Google Sheets (ผ่าน Next.js API Proxy ไปยัง GAS Web App) อย่างไร้รอยต่อ

---

## 1. การเข้าสู่ระบบ (Login)
- **สิทธิ์การใช้งาน (Roles):**
    - **ADMIN / QSMS (ผู้ดูแลระบบ):** เข้าถึงได้ทุกเมนู รวมถึงแดชบอร์ดสถิติ การจัดการ Roster ตารางเวร และโหมดแก้ไขข้อมูลใบงาน (Edit Mode) ในหน้ารายละเอียดเคส
    - **OPERATOR / WFG / PDB (สิทธิ์ปฏิบัติการ Rework):** กลุ่มผลิตและคลังสินค้า โดยจำกัดการใช้งาน **เฉพาะโมดูล Rework** เท่านั้น (ซ่อนโมดูล Roster ทั้งหมด)
        - สามารถเพิ่มงานใหม่ (Add Case) ค้นหาและดูภาพรวม (Overall) ได้
        - สามารถใส่รายละเอียดสินค้า ข้อมูลความเสียหาย และอัปโหลดภาพถ่ายประกอบได้
        - สามารถบันทึกและอัปเดตสถานะงานเป็น "กำลังดำเนินการ (In-Progress)" หรือ "รอประเมินราคา (Awaiting Valuation)"
        - **ข้อจำกัด:** ถูกซ่อนและล็อกปุ่มแก้ไขค่าใช้จ่ายราคาทั้งหมด (No Pricing/Cost Access) รวมถึงไม่มีสิทธิ์ทำการ Export ข้อมูลหรือเรียกดูหน้าแดชบอร์ด
    - **FINANCE (แผนกการเงิน):** จำกัดสิทธิ์เฉพาะการดูตารางภาพรวม (Overall) และทำการกรอกประเมินราคางาน (Rework Cost และ Labor Rate) ในช่องใบงาน เพื่อส่งมอบงานเป็นขั้นตอนเสร็จสมบูรณ์ (Completed) แต่จะไม่สามารถกดเพิ่มงาน (Add Case) หรือแก้ไขรายละเอียดสินค้าด้านในได้
- **วิธีการล็อกอิน:** กรอกรหัสผู้ใช้งานและรหัสผ่านจากระบบกลาง หรือใช้ **บัญชีสำหรับการทดสอบภายใน (Mock Accounts)** ดังนี้:
    - **บัญชี QSMS/Admin:** Username: `qsms`, Password: `Qsms123`
    - **บัญชี Operator/WFG/PDB:** Username: `operator`, Password: `Operator123`
    - **บัญชี Finance:** Username: `finance`, Password: `Finance123`

## 2. หน้าหลัก: ภาพรวมรายการ (Overall)
หน้านี้เปรียบเสมือนศูนย์กลางการติดตามงานทั้งหมด
- **การทักทาย:** ระบบจะทักทายตาม Role ที่ Login เข้ามา (เช่น "สวัสดี QSMS")
- **การค้นหาและกรอง:**
    - ค้นหาด้วยคำสำคัญ (Search) เช่น รหัสสินค้า ชนิดอาการ หรือหมายเลข Case ID
    - กรองตามสถานะ (Pending, In-Progress, Awaiting Valuation, Completed)
    - กรองขั้นสูง (Advanced Filter) ตามแหล่งที่มา, สาเหตุ, ผู้รับผิดชอบ และช่วงวันที่
- **สรุปสถิติเบื้องต้น:** แสดงจำนวนงานในแต่ละสถานะที่ด้านบนของหน้า
- **การจัดเรียง (Sorting):** รายการเคสที่มีสถานะ `Pending` (รอดำเนินการ) จะแสดงขึ้นด้านบนสุดเสมอ เพื่อป้องกันไม่ให้งานคงค้างถูกดันตกไปด้านล่าง จากนั้นจึงตามด้วย `In-Progress`, `Awaiting Valuation` และ `Completed` โดยจัดเรียงตามลำดับวันที่ล่าสุดลงไป

## 3. การเพิ่มงานใหม่ (Add Case)
ใช้สำหรับลงบันทึกงาน Rework ที่เกิดขึ้นใหม่
- **ข้อมูลพื้นฐาน:** เลือกแหล่งที่มาของงาน (Source) ได้แก่ SFC หรือ Customer
- **หมายเลขเคส (Case ID):** กรอกหมายเลขรันนิ่ง (เช่น 012) ระบบจะประกอบเป็น Case ID อัตโนมัติ (เช่น `RW012-2026` หรือ `RT012-2026`) และตรวจสอบความซ้ำซ้อนทันที
- **ข้อมูลสินค้า (Items):** สามารถเพิ่มได้หลายรายการ (Multi-Item) ในหนึ่ง Case
    - **Customer Name:** เลือกชื่อลูกค้าจากตัวเลือก (เช่น OR, Eneos, BCP)
    - **Item Number / Item Code:** ป้อนรหัสสินค้าหรือบาร์โค้ด ระบบจะค้นหาและดึงชื่อสินค้ามาเติมให้อัตโนมัติ (Two-Way Autofill)
    - **Granular Fields (ข้อมูลล็อตสินค้า):** 
        - *หมายเลขล็อต (Batch no.)* และ *วันที่ผลิตแกลลอน (Gallon Date)*: เลือกจากปฏิทินวันที่
        - *เลขกล่อง*, *Mold (หมายเลขแม่พิมพ์)*, และ *Line (สายการผลิต)*: ป้อนข้อมูลระบุพิกัดชิ้นส่วน
    - **Cross-Item Link (เชื่อมโยงสินค้า):** ในกรณีที่สินค้าชิ้นนั้นพบคราบสกปรก ("เปื้อน") และในเคสเดียวกันมีสินค้าชิ้นอื่นที่พบคราบน้ำมันไหล ("รั่ว") ระบบจะเปิดกล่องเช็คบอกซ์พิเศษให้ติ๊กเลือกเชื่อมโยงว่า "สาเหตุมาจากไอเทมที่รั่วในเคสนี้" และระบุไอเทมต้นเหตุได้โดยตรง
    - **เอกสารพิเศษสำหรับ OR (OR Documents):** หากทุกไอเทมในเคสเลือกแบรนด์ลูกค้าเป็น "OR" ระบบจะเปิดช่องพิเศษด้านบนของฟอร์มให้แนบไฟล์เอกสารอ้างอิง (.xlsx, .xls, .pdf, .png) ได้สูงสุด 2 ไฟล์ หากไม่ได้อัปโหลด ระบบจะแสดง Badge เตือน "ขาดไฟล์ OR" สีแดงบนหน้าตารางภาพรวมหลัก
- **รูปภาพหลักฐาน (Evidence Integrity):** ทุกๆ รายการสินค้าจำเป็นต้องมีการแนบรูปภาพอย่างน้อย 1 รูป ระบบจะบีบอัดรูปภาพฝั่ง Client อัตโนมัติ (เป้าหมาย 300KB) เพื่อให้การบันทึกข้อมูลและอัปโหลดรูปภาพทั้งหมดสำเร็จร่วมกันเป็นธุรกรรมเดียว (Transaction Integrity)

## 4. การจัดการและอัปเดตรายการ (Update & Manage)
เมื่อคลิกที่รายการในหน้า Overall จะเปิดหน้าต่างแก้ไข (Update Modal)
- **การอัปเดตสถานะ (Status Lifecycle):** เปลี่ยนสถานะงานตามความคืบหน้า (รอดำเนินการ ➡️ กำลังดำเนินการ ➡️ รอประเมินราคา ➡️ เสร็จสิ้น) โดยระบบมีปุ่มลัดสตรีมมิ่งเปลี่ยนสถานะด่วนในแถบด้านล่างของหน้าต่าง
- **การระบุวิธีแก้ไขและทรัพยากร:**
    - **วิธีแก้ไข:** ใส่รายละเอียดการแก้ไขในช่อง "Resolution Method"
    - **ชิ้นส่วน/วัสดุเบิกใช้ (Materials):** ผู้ใช้สามารถกดเพิ่มวัสดุ เช่น ฝา, สติ๊กเกอร์, แกลลอน พร้อมระบุจำนวนและราคาต่อหน่วย
    - **ชั่วโมงและกำลังพล (Labor):** กรอกจำนวนคนทำงาน (Labor Count) และชั่วโมงการทำงาน (Labor Hours) เพื่อให้ระบบนำไปคำนวณต้นทุนรวม
- **การประเมินค่าใช้จ่าย (Finance/Admin):** ระบบจะคำนวณราคารวมของวัสดุและราคาค่าแรง (ตาม Labor Rate ที่การเงินระบุ) ออกมาเป็นยอดเงินจริง "Rework Cost" ให้โดยอัตโนมัติ
- **โหมดแก้ไขของผู้ดูแล (Administrative Edit Mode):** (สำหรับ Admin เท่านั้น) สามารถกดปุ่ม "แก้ไขข้อมูล" เพื่อเปลี่ยนรายละเอียดดั้งเดิมของใบงาน เช่น แหล่งที่มา, หมายเลขใบงาน, ข้อมูลฟิลด์ต่างๆ ของสินค้า, ลบรายการสินค้าบางชิ้น, หรือเพิ่มรูปถ่ายเพิ่มเติมได้ โดยเมื่ออยู่ในโหมดแก้ไข หากเผลอกดยกเลิก ระบบจะแจ้งเตือนยืนยัน "คุณมีข้อมูลที่ยังไม่ได้บันทึก ต้องการปิดใช่หรือไม่?" เสมอเพื่อป้องกันข้อมูลหาย

## 5. การออกรายงาน (Export Report)
ภายในหน้าต่าง Update Modal จะมีเมนูส่งออกข้อมูลสำหรับจัดพิมพ์หรือส่งต่อ:
- **PDF Export:** ส่งออกใบงาน rework ออกเป็นฟอร์แมตเอกสารทางการที่สวยงาม พร้อมจัดเรียงข้อมูลและรูปภาพหลักฐานเรียบร้อย
- **Excel Export:** ส่งออกข้อมูล rework ทั้งหมดออกเป็นไฟล์สเปรดชีตสากล (.xlsx) ด้วยไลบรารี `exceljs` ที่มาพร้อมคุณสมบัติระดับสูง:
    - ตกแต่งสีสันหัวตารางเป็นสากล (แถบสีน้ำเงินอักษรขาวหนา)
    - เปิดการห่อหุ้มข้อความแบบขึ้นบรรทัดใหม่อัตโนมัติ (Wrap Text)
    - **ฝังรูปภาพหลักฐานจริง (Image Embedding):** ระบบจะดึงรูปหลักฐานที่อัปโหลดไว้ (สูงสุด 3 รูปแรก) มาแปลงเป็น base64 และฝังลงในคอลัมน์ "Images" ของแต่ละบรรทัดสินค้าโดยตรง พร้อมปรับความสูงของแถวข้อมูลเป็น 120px เพื่อจัดวางภาพถ่ายหลักฐานได้อย่างสวยงามและสมบูรณ์

## 6. แดชบอร์ดวิเคราะห์ผล (Dashboard)
(เฉพาะผู้ใช้ระดับ Admin และ QSMS)
- **KPI Summary:** ดูภาพรวมประสิทธิภาพการทำงาน เปอร์เซ็นต์งานที่เสร็จสิ้น และเคสที่ตกค้างเกินกำหนด (แจ้งเตือนงานค้างเกิน 7 วัน และ 30 วันสีส้มและสีแดง)
- **Chart Analysis:** แผนภูมิกราฟวิเคราะห์สาเหตุความบกพร่องยอดฮิต (Defect Reasons) และสัดส่วนภาระงานแบ่งตามแหล่งที่มา

## 7. ระบบการทดสอบอัตโนมัติ (Automated Testing Pipeline)
สำหรับนักพัฒนาหรือผู้ดูแลระบบที่ต้องการตรวจสอบความเสถียรของแอปพลิเคชันก่อนนำไปใช้งานจริง:
- **Unit & Integration Tests (Vitest):** ใช้ตรวจสอบการทำงานของ Helpers, ฟังก์ชันการจัดเรียงสถานะ และฟังก์ชันการทำงานของโมดอลแก้ไขข้อมูล
  * คำสั่งรัน: `npm run test`
- **End-to-End (E2E) Tests (Playwright):** จำลองการกดเบราว์เซอร์จริง เช่น ล็อกอิน, กรอกฟอร์ม Rework, เลือกโมดอล และตรวจเช็คผลการบันทึกข้อมูล
  * คำสั่งรัน: `npm run test:e2e`
  * ดีบั๊กแบบ Interactive UI: `npx playwright test --ui`

## 8. ระบบถามตอบอัจฉริยะ (DocAI RAG - น้องผึ้งพา)
แผงปุ่มแชทอัจฉริยะในหน้าจอคลังแอปพลิเคชัน (DocAI) สำหรับช่วยสืบค้นคู่มือโรงงานและแนวทางการซ่อมบำรุง:
- **Nong Beepa Persona:** บอทแชทภาษาไทยที่เป็นมิตร สุภาพ มีภาพลักษณ์แบรนด์ที่น่ารัก เพื่อลดความตึงเครียดของพนักงานบนพื้นที่ทำงานจริง
- **การนำเข้าคลังความรู้ (Ingestion Pipeline):**
    - ผู้ใช้สามารถเข้าแท็บ "เอกสาร" เพื่ออัปโหลดไฟล์คู่มือ (.pdf, .xlsx, .xls, .jpg, .png)
    - สำหรับไฟล์ PDF ระบบเบื้องหลังจะใช้ `pdfjs-dist` แปลงหน้าเอกสารเป็นรูปภาพทีละหน้า อัปโหลดขึ้น Supabase Storage และส่งต่อให้ `gemini-2.5-flash` อ่านรูปภาพและแผนผังคู่มืออย่างละเอียดถอดเป็น Markdown ออกมา
    - ข้อความ Markdown จะถูกแบ่งสัดส่วนเนื้อหาเชิงความหมาย (Semantic Chunking) และแปลงเป็นเวกเตอร์ 768 มิติด้วย `jina-embeddings-v5-text-small` เพื่อบันทึกเข้า Supabase pgvector
- **การจัดการความรู้เป็นกลุ่ม (Bulk Delete):** ในแท็บเอกสาร ผู้ใช้สามารถทำเครื่องหมายติ๊กเลือก (Checklist) หน้าไฟล์เอกสารหลายตัวพร้อมกัน และกดปุ่มลบแบบกลุ่ม ("ลบที่เลือก") เพื่อล้างข้อมูล metadata และลบชิ้นส่วนเวกเตอร์ที่เกี่ยวข้องทั้งหมดออกจากระบบได้อย่างรวดเร็ว
- **การถามตอบ (Hybrid Chat):** เมื่อผู้ใช้พิมพ์คำถาม ระบบจะใช้ Hybrid Search ค้นหาด้วยความใกล้เคียงของเวกเตอร์คู่กับคำค้นหาตรงตัว นำบริบทที่ดีที่สุด 5 แหล่งมาประมวลผลเป็นคำตอบพร้อมสตรีมมิ่งข้อความสด หากคู่มือนั้นมีแผนภาพหรือรูปถ่าย ระบบจะแสดงรูปภาพประกอบในหน้าแชทบอทให้ผู้ใช้คลิกเปิดดูรูปขยาย (Lightbox) ได้ทันที
- **คำถามแนะนำและระบบ Feedback:** บอทจะแนะนำคำถาม 3 ข้อถัดไปที่เกี่ยวข้อง (Suggestion Chips) และผู้ใช้งานสามารถคลิกให้คะแนน Thumbs Up/Down แก่คำตอบนั้นๆ เพื่อบันทึกเข้าตาราง `rag_feedback` ของระบบ
